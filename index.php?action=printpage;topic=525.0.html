When starting, cs2d occasionally tells you that cheating/hacking is forbidden. If a player is dumb enough to ignore this, he deserves to be banned straight away. No need to warn hackers, no need to speclock them, just be 100% sure that they're actually hacking before letting the ban hammer fall.

Also there is a huge range of how people spam/insult each other, so I think moderators should be expected to be able to handle each case properly without such guidelines.
E.g. compared to seven ":D"s, which some of you might consider spamming, "@@@@@@@@@@@@@@@@@@" is a completely different dimension of spamming which has to be treated differently ( / smileys just smile, @@@@@@..s block the whole screen). While a warning should be enough for the smiley-guy, the "@@@@@@@@@@.."-player should be muted (/kicked) instantly if he did it repetitively.
